# ドキュメント目次

このディレクトリには、 `Yu-Gi-Oh! ソリティア` の**ストック情報**（永続的な知識）が整理されています。

---

## 💡 プロジェクトコンセプト

### 課題と目的

**課題**:

- 遊戯王の「ソリティア（1 人で延々とコンボを回す行為）」は、実カードの収集や練習が煩雑
- 自分は楽しいが、対戦相手はつまらない（壁とやってろ）
- 特定のデッキで勝利までのルートに挑戦したいニーズがある
- 過去に猛威を振るったワンキルデッキは、主要パーツのエラッタ・禁止により再現が難しい

**目的**:

- Web ブラウザ上で手軽に「先攻 1 ターンキル」に挑戦・練習できる環境を提供する

### コンセプト

**ソリティア感覚でプレイする、 1 人プレイ用の遊戯王 1 ターンキル・シミュレーター**

- ユーザーはデッキを選ぶだけで、即座にゲームを開始できる
- 「全盛期」のカード・ワンキルデッキを体験できる
- 対戦相手はカカシで、気兼ねも思考待ち時間もゼロ、優先権も考慮不要
- 先行 1 ターン目に絞ることで、複雑な戦闘ダメージ計算等の実装は不要
- カードプールを絞ることで、大量かつ複雑な実装は不要

---

## 🔍 スコープ方針

### ✅ やること (In Scope)

- **先行 1 ターン目のゲーム進行**: 1 キルシミュレーターに必要な範囲に絞ったゲーム進行管理システム
- **プリセットデッキ選択**: プレイヤーは、あらかじめ定義されたデッキから選択
- **プリセットカードプール**: プリセットデッキに含まれるカードプールに必要なカードの効果処理のみ実装
- **勝利判定**: ライフポイント、デッキ切れ、またはプリセットデッキで達成可能な特殊勝利

### ❌ やらないこと (Out of Scope)

- **デッキ構築機能**: カードプール爆発の防止
- **バトルフェイズ**: 先攻 1 ターン目には不要
- **相手プレイヤー**: 基本的には何もしないカカシ（必要に応じて、ランダム選択等の軽量ルールベース処理）
- **優先権システム**: 相手プレイヤーが何もしないため、不要

## 🗂️ ドキュメント構成

```
# ストック情報（永続的な情報）
docs/
├── domain/                    # ドメイン知識: ゲームルールと実装のマッピング
├── architecture/              # アーキテクチャ: 技術的な構成・方針
└── adr/                       # ADR: 設計判断の記録

# フロー情報（spec-kit による作業記録）
specs/
└── 001-architecture-refactoring/  # 1ショットの作業単位
    ├── spec.md                # この作業単位の要件
    ├── plan.md                # この作業単位の計画
    └── tasks.md               # この作業単位の進捗
```

### ドキュメント更新が必要になるタイミング

| ドキュメント      | 更新タイミング                               |
| ----------------- | -------------------------------------------- |
| **domain/**       | 新しいゲームルールを検討する時 or 実装した時 |
| **architecture/** | アーキテクチャを大きく変更した時             |
| **adr/**          | 重要な技術的決定をした時                     |

## 🔧 技術スタック

### フロントエンド

- **フレームワーク**: SvelteKit + Svelte 5
- **UI ライブラリ**: Skeleton UI v3
- **CSS**: TailwindCSS v4
- **状態管理**: Svelte Stores + Immer.js（不変性保証）
- **型チェック**: TypeScript (Strict mode)
- **テスト**: Vitest（Unit tests / Integration tests） + Playwright（E2E tests）
