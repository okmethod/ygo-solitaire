# ドメイン知識

「遊戯王のルール」そのものを記述した、フレームワークやライブラリに依存しない **純粋な知識** を整理する。  
また、それらのコードベースとの対応関係と、実装状況を一覧化する。

---

## 📄 ファイル一覧

### ルール概要: [overview.md](./overview.md)

**内容**: ゲームルール全体像と実装状況のマッピング

- ドメイン知識とコードの対応表
- ドメイン実装状況（✅ 完全実装 / 🚧 一部実装 / ⏳ 未実装 / ❌ 実装不要）

**読むべき人**: 全員（最初に読むドキュメント）

### カードモデル: [card-model.md](./card-model.md)

**内容**: カードのデータ構造とモデル定義

- カード種別（モンスター・魔法・罠）
- カード種別ごとのプロパティ
- カードデータとカード実体の違い
- カードの位置・表示形式

**読むべき人**: カード実装に関わる開発者

### 効果モデル: [effect-model.md](./effect-model.md)

**内容**: カード効果の処理システム

- CardEffect Interface
- CardEffectRegistry（Strategy Pattern）
- 実装済みカード効果一覧

**読むべき人**: カード効果実装に関わる開発者

### ゲーム操作モデル: [game-operations.md](./game-operations.md)

**内容**: ゲームの状態を変化させる手続きの定義

- プレイヤーが選択可能な行動がトリガーになるもの（召喚、セット、発動、等）
- ルールに従い自動的に処理されるもの（フェイズ進行、ドロー、勝利判定、等）
- 各操作の実装状況

**読むべき人**: ゲームロジック実装に関わる開発者

### チェーンシステム: [chain-system.md](./chain-system.md)

**内容**: 効果の連鎖処理システム

- チェーンの概念（LIFO 処理）
- ChainStack の実装
- チェーン解決の仕組み

**読むべき人**: 高度なカード効果実装に関わる開発者

### 特殊勝利条件: [victory-conditions.md](./victory-conditions.md)

**内容**: 特殊な勝利条件の詳細

- ライフポイント切れ・デッキ切れ以外の、各種カードの効果による勝利条件

**読むべき人**: 勝利判定ロジック実装に関わる開発者

---

## 📖 読む順序

1. **新規参加者**: まず [overview.md](./overview.md) でルールの全体像と実装状況を把握
2. **実装開始前**: 改修箇所に応じて、各種ドメイン情報のルールを把握
