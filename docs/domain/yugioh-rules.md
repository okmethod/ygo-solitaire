# 遊戯王OCG基本ルール

本アプリケーションで扱う遊戯王OCG（オフィシャルカードゲーム）の基本ルールを定義します。

---

## ユビキタス言語 (Ubiquitous Language)

開発チームとドメイン（カードゲーム）の共通言語。

### 領域 (Zone)

| Zone | 日本語 | 説明 |
|------|--------|------|
| **Deck** | デッキ | 山札。ここからカードを引く |
| **Hand** | 手札 | プレイヤーが持っているカード。プレイの起点 |
| **Field** | フィールド | モンスターゾーン、魔法・罠ゾーンの総称 |
| **Graveyard** | 墓地 | 使用済みのカードが送られる場所 |
| **Banishment** | 除外 | ゲームから除外された場所 |

### 行動 (Action)

| Action | 日本語 | 説明 |
|--------|--------|------|
| **Draw** | ドロー | デッキの一番上からカードを手札に加える |
| **Summon** | 召喚 | モンスターをフィールドに出す |
| **Activate** | 発動 | 魔法カードを使用する、またはモンスター効果を使用する |
| **Set** | セット | カードを裏側表示でフィールドに置く |
| **Release** | リリース | コストとしてモンスターを墓地へ送る |

### 概念 (Concept)

- **Chain (チェーン)**: 効果の発動に対して、別の効果を発動すること。LIFO（後入れ先出し）で解決される
- **Cost (コスト)**: 効果を発動するために支払う代償（ライフ、手札捨て等）。効果無効になっても返ってこない
- **Effect (効果)**: カードが持つ固有の処理内容

## フェイズシステム

先攻1ターン目のフェイズ遷移：

```
Draw Phase (ドローフェイズ)
  ↓
Standby Phase (スタンバイフェイズ)
  ↓
Main Phase 1 (メインフェイズ1)
  ↓
End Phase (エンドフェイズ)
```

### 各フェイズの説明

#### Draw Phase
- デッキの一番上からカードを1枚ドローする
- 先攻1ターン目は初期手札配布済みのため、ドローしない

#### Standby Phase
- 特定のカード効果が発動するフェイズ
- 本アプリではスキップ可能

#### Main Phase 1
- **カード発動**: 魔法カードを発動できる
- **モンスター召喚**: モンスターを召喚できる
- **カードセット**: カードを伏せて配置できる

#### End Phase
- ターン終了時の処理を行う
- 本アプリでは1ターン目限定のため、ここでゲーム終了

## 勝利条件

### 1. エクゾディア勝利
**条件**: 以下の5枚すべてが手札に揃う
- 封印されしエクゾディア
- 封印されし者の右足
- 封印されし者の左足
- 封印されし者の右腕
- 封印されし者の左腕

**判定タイミング**: 任意のタイミング（ドロー直後など）

### 2. ライフポイント勝利
**条件**: 相手のライフポイントを0にする

**初期LP**: 8000

**本アプリでの扱い**: バーンダメージによる勝利を実装

## カード種別

### Monster (モンスターカード)
- フィールドに召喚して使用
- 攻撃力・守備力を持つ
- **本アプリでのスコープ**: エクゾディアパーツのみ実装

### Spell (魔法カード)
- 手札から発動
- 発動後は墓地へ送られる
- **Normal Spell (通常魔法)**: メインフェイズ1に発動可能

### Trap (罠カード)
- セットして次のターン以降に発動可能
- **本アプリでのスコープ**: 発動判定ルールのみ実装（先攻1ターン目は発動不可、手札からの直接発動も不可）

---

## アクター (Actor)

- **Player (プレイヤー)**: ゲームを操作する主体
- **System (システム/ジャッジ)**: ルールを監視し、処理を実行する主体。相手プレイヤーの代わりも兼ねる
